发现网上-g --save-dev等位置有前有后，稍微规范点没什么不好，以下部分内容摘录官网

  • 官方手册 https://www.npmjs.com.cn/
  • 安装本地包 npm install <package_name> 在本地目录中如果没有 package.json 这个文件的话，那么最新版本的包会被安装。
  • 卸载本地包 npm uninstall <package_name> 如从 package.json中删除依赖则npm uninstall --save lodash。
  • 更新本地包 npm update 在 package.json 所在的目录中执行该命令。
  • 安装全局包 npm install -g <package_name>
  • 卸载全局包 npm uninstall -g <package_name>
  • 更新全局包 npm update -g <package_name>
  • 查看帮助 1、npm -h   2、npm i -h
  • 查看升级版本 1、npm -v   2、npm install npm@latest -g
  • 注册淘宝镜像 npm install -g cnpm --registry= https://registry.npm.taobao.org
  • 参数-P, --save-prod Package will appear in your dependencies. This is the default unless -D or -O are present.
  • 参数-D, --save-dev Package will appear in your devDependencies. npm install node-tap --save-dev
